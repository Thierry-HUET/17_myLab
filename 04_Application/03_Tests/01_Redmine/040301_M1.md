## Module M1 — Administration (Jour 1 — 7h)

**Public** : gestionnaire(s) de l'instance  
**Objectif** : installer, configurer et administrer une instance Redmine 5.1.12 prête à l'emploi

---

::::{exercise} EX-01 — Installation Docker Desktop + Redmine
:label: redmine_ex01
:nonumber:

**Durée** : 1h — **Public** : tous (guidé par le formateur)  
**Prérequis** : Windows 10/11 64 bits, droits administrateur, connexion internet

**Étapes**

1. Télécharger et installer [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)
2. Activer WSL2 si demandé par l'installeur — redémarrer si nécessaire
3. Vérifier l'installation

```bash
docker --version
```

4. Créer le dossier `C:\redmine-formation\` et y créer le fichier `docker-compose.yml` :

```yaml
version: '3.8'
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: redmine
      POSTGRES_USER: redmine
      POSTGRES_PASSWORD: redmine
  redmine:
    image: redmine:5.1.12
    depends_on:
      - db
    ports:
      - "3000:3000"
    environment:
      REDMINE_DB_POSTGRES: db
      REDMINE_DB_DATABASE: redmine
      REDMINE_DB_USERNAME: redmine
      REDMINE_DB_PASSWORD: redmine
```

5. Lancer l'instance depuis PowerShell

```bash
docker compose up -d
```

6. Ouvrir un navigateur → `http://localhost:3000`
7. Se connecter : `admin` / `admin` — changer le mot de passe à la première connexion

**Critère de réussite** : interface Redmine accessible, connexion admin opérationnelle

::::{exercise} EX-04 — Définition des statuts et workflow
:label: redmine_ex04
:nonumber:

**Durée** : 45 min  
**Menu** : Administration → Statuts des demandes, puis Workflow

**Statuts à créer** :

| Statut | Statut fermé |
|---|---|
| Nouveau (défaut à la création) | non |
| En cours | non |
| En attente client | non |
| Résolu | non |
| Fermé | **oui** |
| Rejeté | **oui** |

**Workflow à configurer** — Tracker : Anomalie / Rôle : Développeur :

| Depuis | Vers |
|---|---|
| Nouveau | En cours, Rejeté |
| En cours | En attente client, Résolu |
| En attente client | En cours, Fermé |
| Résolu | Fermé, En cours |

**Critère de réussite** : matrice workflow affiche les transitions sans erreur
::::

::::{exercise} EX-05 — Création des rôles et permissions
:label: redmine_ex05
:nonumber:

**Durée** : 30 min  
**Menu** : Administration → Rôles et permissions

| Rôle | Permissions clés |
|---|---|
| Chef de projet | Gérer membres, consulter tous les temps, modifier toutes les demandes |
| Développeur | Créer/modifier ses demandes, saisir ses temps |

**Critère de réussite** : 3 rôles distincts avec permissions différenciées

## Livrable M1 — Vérification croisée

**Durée** : 30 min

Chaque stagiaire joue le rôle d'un auditeur et vérifie l'instance d'un autre à l'aide de la check-list :

- [ ] 4 trackers présents et bien nommés
- [ ] Workflow Anomalie / Développeur fonctionnel
- [ ] 3 rôles avec permissions distinctes
- [ ] 3 comptes actifs
- [ ] 5 activités de temps configurées

**Résultat attendu** : check-list complète à vert — instance prête pour M2 et M3
