::::{exercise} EX-01 — Installation Docker Desktop + Redmine
:label: redmine_ex01
:nonumber:

**Durée** : 45 min — **Public** : tous (guidé par le formateur)  
**Prérequis** : Windows 10/11 64 bits, droits administrateur, connexion internet

**Étapes**

1. Télécharger et installer [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop)
2. Activer WSL2 si demandé par l'installeur — redémarrer si nécessaire
3. Vérifier l'installation

```bash
docker --version
```

4. Créer le dossier `C:\redmine-formation\` et y créer le fichier `docker-compose.yml` :

```yaml
version: '3.8'
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: redmine
      POSTGRES_USER: redmine
      POSTGRES_PASSWORD: redmine
  redmine:
    image: redmine:5.1.12
    depends_on:
      - db
    ports:
      - "3000:3000"
    environment:
      REDMINE_DB_POSTGRES: db
      REDMINE_DB_DATABASE: redmine
      REDMINE_DB_USERNAME: redmine
      REDMINE_DB_PASSWORD: redmine
```

5. Lancer l'instance depuis PowerShell

```bash
docker compose up -d
```

6. Ouvrir un navigateur → `http://localhost:3000`
7. Se connecter : `admin` / `admin` — changer le mot de passe à la première connexion

**Critère de réussite** : interface Redmine accessible, connexion admin opérationnelle
::::

:label: sol_redmine_ex01
:class: dropdown

Vérifier que les deux conteneurs sont en cours d'exécution :

```powershell
docker ps
```

```
CONTAINER ID   IMAGE            STATUS         PORTS
xxxxxxxxxxxx   redmine:5.1.12   Up X minutes   0.0.0.0:3000->3000/tcp
xxxxxxxxxxxx   postgres:15      Up X minutes
```

En cas d'erreur, consulter les logs :

```powershell
docker compose logs redmine
docker compose logs db
```

Erreurs fréquentes :
- Port 3000 déjà utilisé → modifier `"3000:3000"` en `"3001:3000"` dans le `docker-compose.yml`
- WSL2 non activé → Panneau de configuration → Activer les fonctionnalités Windows → Sous-système Windows pour Linux

::::{exercise} EX-08-ADV — Workflow multi-rôles et multi-trackers
:label: redmine_ex08_adv
:nonumber:

**Durée** : 45 min  
**Menu** : Administration → Workflow

EX-04 n'a configuré le workflow que pour le tracker Anomalie et le rôle Développeur. L'objectif est ici d'étendre la matrice à l'ensemble des trackers et des rôles, en raisonnant sur les droits métier de chaque acteur.

**Matrice à compléter** :

| Tracker | Rôle | Transitions autorisées |
|---|---|---|
| Évolution | Chef de projet | Nouveau → En cours, Rejeté / En cours → Résolu, En attente client / Résolu → Fermé |
| Évolution | Développeur | Nouveau → En cours / En cours → En attente client, Résolu |
| Évolution | Client | Résolu → Fermé uniquement |
| Demande de service | Chef de projet | Toutes les transitions |
| Demande de service | Développeur | Nouveau → En cours / En cours → Résolu, En attente client |
| Demande de service | Client | Résolu → Fermé uniquement |
| Tâche interne | Chef de projet | Toutes les transitions |
| Tâche interne | Développeur | Nouveau → En cours / En cours → Résolu |
| Tâche interne | Client | Aucune transition (lecture seule) |

**Contraintes à respecter** :
- Un Client ne peut jamais passer une demande en "Rejeté"
- Un Client ne peut jamais rouvrir une demande fermée
- Seul le Chef de projet peut passer en "Rejeté" sur tous les trackers

**Vérification** : se connecter avec chaque compte (`cda.dupont`, `dev.martin`, `client.durand`) et contrôler que les transitions disponibles dans l'interface correspondent exactement à la matrice.

**Critère de réussite** : aucune transition non autorisée accessible depuis l'interface, quel que soit le compte connecté
::::

::::{solution} redmine_ex08_adv
:label: sol_redmine_ex08_adv
:class: dropdown

Répéter l'opération pour chaque combinaison Tracker / Rôle :  
Administration → Workflow → sélectionner le tracker et le rôle → Modifier

**Évolution / Client** — seule case à cocher :

| | Fermé |
|---|---|
| Résolu | ✅ |

**Tâche interne / Client** — aucune case cochée (matrice vide).

**Astuce** : utiliser le bouton **Copier** du workflow pour dupliquer la configuration d'un rôle vers un autre et n'ajuster que les différences.

Vérification rapide sans changer de session : Administration → Workflow → sélectionner le tracker et le rôle → **Consulter** (mode lecture seule).

::::{exercise} EX-09-ADV — API REST : interrogation et manipulation via curl
:label: redmine_ex09_adv
:nonumber:

**Durée** : 45 min

**Installation de curl sur Windows**

curl est inclus nativement dans Windows 10/11. Vérifier depuis PowerShell :

```powershell
curl --version
```

Si absent (versions antérieures), télécharger depuis [curl.se/windows](https://curl.se/windows) et ajouter le dossier `bin\` au PATH système.

**Obtention de la clé API**

1. Se connecter à Redmine avec le compte `admin`
2. Menu : Mon compte (en haut à droite) → onglet **Clé d'accès API**
3. Cliquer sur **Réinitialiser** pour générer la clé
4. Copier la clé — elle sera utilisée dans toutes les requêtes via l'en-tête `X-Redmine-API-Key`

:::{note}
La clé API est équivalente à un mot de passe. Ne pas la versionner ni la transmettre en clair.
:::

**Activer l'accès API**

Menu : Administration → Configuration → onglet **API**
- Cocher **Activer l'API REST**
- Sauvegarder

---

**Exemple complet — cycle de vie d'une demande par API**

Remplacer `VOTRE_CLE_API` par la clé obtenue ci-dessus.

*1. Lister les projets disponibles*

```powershell
curl -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     -H "Content-Type: application/json" `
     http://localhost:3000/projects.json
```

Résultat attendu : liste JSON des projets avec leur `id` et leur `identifier`.

*2. Créer une demande dans le projet `mon-projet`*

```powershell
curl -X POST `
     -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     -H "Content-Type: application/json" `
     -d "{\"issue\": {\"project_id\": \"mon-projet\", \"tracker_id\": 1, \"subject\": \"Anomalie creee via API\", \"priority_id\": 2}}" `
     http://localhost:3000/issues.json
```

Noter l'`id` de la demande créée dans la réponse JSON (ex : `"id": 42`).

*3. Consulter la demande créée*

```powershell
curl -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     http://localhost:3000/issues/42.json
```

*4. Modifier le statut (passage en "En cours" — status_id: 2)*

```powershell
curl -X PUT `
     -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     -H "Content-Type: application/json" `
     -d "{\"issue\": {\"status_id\": 2, \"notes\": \"Prise en charge via API\"}}" `
     http://localhost:3000/issues/42.json
```

*5. Lister les demandes ouvertes du projet*

```powershell
curl -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     "http://localhost:3000/issues.json?project_id=mon-projet&status_id=open"
```

*6. Créer un utilisateur*

```powershell
curl -X POST `
     -H "X-Redmine-API-Key: VOTRE_CLE_API" `
     -H "Content-Type: application/json" `
     -d "{\"user\": {\"login\": \"dev.duval\", \"firstname\": \"Sophie\", \"lastname\": \"Duval\", \"mail\": \"s.duval@example.com\", \"password\": \"Redmine2026!\"}}" `
     http://localhost:3000/users.json
```

**Correspondance status_id / priority_id**

| status_id | Statut | priority_id | Priorité |
|---|---|---|---|
| 1 | Nouveau | 1 | Basse |
| 2 | En cours | 2 | Normale |
| 3 | Résolu | 3 | Haute |
| 4 | Fermé | 4 | Urgente |
| 5 | Rejeté | | |

:::{note}
Les `status_id` dépendent de l'ordre de création des statuts dans votre instance. Vérifier via `http://localhost:3000/issue_statuses.json`
:::

**Critère de réussite** : demande créée, statut modifié et visible dans l'interface web, utilisateur `dev.duval` actif dans Administration → Utilisateurs
::::

::::{solution} redmine_ex09_adv
:label: sol_redmine_ex09_adv
:class: dropdown

**Réponse attendue à l'étape 2 (création de demande)** :

```json
{
  "issue": {
    "id": 42,
    "project": {"id": 1, "name": "Formation Redmine"},
    "tracker": {"id": 1, "name": "Anomalie"},
    "status": {"id": 1, "name": "Nouveau"},
    "priority": {"id": 2, "name": "Normale"},
    "subject": "Anomalie creee via API"
  }

```

- `401 Unauthorized` → clé API incorrecte ou API REST non activée
- `422 Unprocessable Entity` → champ obligatoire manquant (vérifier `tracker_id`, `subject`)
- `404 Not Found` → `project_id` ou `issue id` incorrect

**Récupérer les tracker_id de votre instance** :

```powershell

     http://localhost:3000/trackers.json
```
