## Module M2 — Gestion de projet (Jour 2 matin — 3h30)

**Prérequis** : instance configurée par M1 — rôles, trackers, workflow et activités en place  
**Objectif** : créer et piloter un projet Redmine, gérer ses membres et produire des rapports de temps

---

::::{exercise} EX-08 — Création et structuration du projet
:label: redmine_ex08
:nonumber:

**Durée** : 30 min  
**Menu** : Projets → Nouveau projet

| Champ | Valeur |
|---|---|
| Identifiant | `tma-capgemini` |
| Nom | `TMA Capgemini Engineering` |
| Description | Contexte fictif de maintien en condition opérationnelle |
| Modules à activer | Suivi des demandes, Wiki, Documents, Suivi du temps, Fichiers |

Créer ensuite un sous-projet :
- Identifiant : `tma-capgemini-batch`
- Projet parent : `tma-capgemini`

**Critère de réussite** : projet et sous-projet visibles dans l'arborescence
::::

::::{exercise} EX-09 — Affectation des membres et des rôles
:label: redmine_ex09
:nonumber:

**Durée** : 20 min  
**Menu** : Projet → Paramètres → Membres

| Compte | Rôle |
|---|---|
| `cda.dupont` | Chef de projet |
| `dev.martin` | Développeur |
| `client.durand` | Client |

Vérifier en changeant de compte que chaque utilisateur accède uniquement aux fonctionnalités de son rôle.

**Critère de réussite** : chaque compte accède au projet avec les bonnes restrictions
::::

::::{exercise} EX-11 — Création d'un lot de demandes fil rouge
:label: redmine_ex11
:nonumber:

**Durée** : 45 min  
**Menu** : Projet → Nouveau ticket (répété)

| Tracker | Sujet | Priorité | Assigné à | Version |
|---|---|---|---|---|
| Anomalie | Erreur 500 sur le module facturation | Urgente | dev.martin | Sprint 1 |
| Anomalie | Calcul TVA incorrect sur devis PDF | Haute | dev.martin | Sprint 1 |
| Évolution | Ajout export Excel sur liste commandes | Normale | — | Sprint 2 |
| Demande de service | Réinitialisation mot de passe utilisateur Dupont | Normale | dev.martin | Sprint 1 |
| Tâche interne | Mise à jour documentation API | Basse | dev.martin | Sprint 2 |
| Anomalie | Timeout connexion base de données | Urgente | dev.martin | Sprint 1 |

**Critère de réussite** : 6 demandes visibles, filtrables par tracker et par version
::::

::::{exercise} EX-12 — Mise en place de la documentation projet (Wiki)
:label: redmine_ex12
:nonumber:

**Durée** : 30 min  
**Menu** : Projet → Wiki

Créer les pages suivantes :

| Page | Contenu attendu |
|---|---|
| Accueil | Présentation du projet, contacts, liens utiles |
| Procedure-Incident | Procédure de qualification d'une anomalie (3 étapes minimum) |
| Contacts | Tableau fictif des interlocuteurs (nom, rôle, email) |

Depuis la page Accueil, créer un lien vers la procédure :

```
[[Procedure-Incident]]
```

**Critère de réussite** : wiki navigable, lien fonctionnel entre les pages
::::

---

::::{exercise} EX-13 — Consultation et analyse des temps (simulation)
:label: redmine_ex13
:nonumber:

**Durée** : 30 min

**Saisie des temps** — chaque stagiaire saisit 3 entrées :

| Demande | Durée | Activité |
|---|---|---|
| Erreur 500 sur le module facturation | 2h | Développement |
| Réinitialisation mot de passe | 0,5h | Support / Assistance |
| Mise à jour documentation API | 1h | Documentation |

**Analyse** — Menu : Projet → Suivi du temps

- Filtrer par activité
- Filtrer par utilisateur
- Filtrer par version
- Identifier le total d'heures sur Sprint 1

**Critère de réussite** : rapport affiche les temps saisis, tous les filtres fonctionnels
